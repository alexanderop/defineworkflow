Claude Code is not the conductor

The workflow engine decides when an agent exists. At an agent() call, the Claude adapter starts a process like: 

claude -p "<prompt>" \
  --output-format stream-json \
  --verbose \
  --dangerously-skip-permissions \
  --add-dir <workflow cwd>

Claude returns streamed JSON. The adapter translates that into agent-output, agent-progress, and agent-finished events. The runtime writes the final result to journal.jsonl so resume can skip it later.

04

How to inspect what happened

Start with the terminal UI. If you want raw proof, inspect the run files. The run directory is the source of truth for watch, resume, and save.

workflow list

Shows known runs with status, elapsed time, and identifiers. Use this when you forgot the run id.

workflow watch <runId>

Attaches the terminal dashboard to a detached or finished run by replaying its event stream.

~/.workflow/runs/<runId>/events.jsonl

Every observable moment: run-started, agent-queued, agent-started, agent-output, agent-finished, failures, logs, and final status.

~/.workflow/runs/<runId>/journal.jsonl

Completed agent results keyed by sequence number. Resume reads this before spawning Claude, so cached agents do not run again.

workflow resume <runId>

Replays journal hits instantly, then spawns Claude only for the first uncached or changed agent() call onward.
