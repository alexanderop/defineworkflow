# `workflow init` — Template Gallery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: executing-plans (inline). Steps use checkbox (`- [ ]`) syntax.

**Goal:** Add `workflow init <template>` and `workflow list-templates` — a bundled, offline, zero-token template gallery that scaffolds an editable `.workflow.ts`, auto-adapts the harness to one installed, and offers a mock first-run.

**Architecture:** Additive `case` strings on the existing pure `dispatch(argv, deps)` router. Templates ship inside the published `defineworkflow` package at `packages/workflow/templates/` (committed + regenerated by a `prebuild` script), surfaced via a new `Env.templatesDir`. A new `templates.ts` (zod schema + `loadTemplateIndex(): Result`) is the single read path. `init` is the read+write subset of the run pipeline; its headline mock-run reuses `runCommand({ mock: true })`. Errors are neverthrow `Result<T,string>` → printed + exit 1, never thrown across the command boundary.

**Tech Stack:** TypeScript (strict, ESM, neverthrow, zod, type-fest `Immutable`), esbuild (template validation), vitest + `fakeDeps`/`memFs` fakes.

**Key constraints discovered:**

- Published layout: `defineworkflow` bundles its own CLI at `packages/workflow/dist/cli.js`; `templatesDir = path.resolve(dirname(cliPath), "..", "templates")`.
- `packages/workflow/tsconfig.json` `include: ["src"]` → `templates/` is NOT typechecked (template files self-import `"defineworkflow"`, which only resolves on a user's disk). Keep templates out of `src`.
- knip would flag template files as unused → add `templates/**` to knip ignore.
- CI runs lint, typecheck (= build, which triggers `prebuild`), knip, affected unit tests. No live example gate. So `copy-templates.mjs` must be self-contained (esbuild only, no import of built `dist`).
- `FileIO` (`app.ts`) currently has only `readText`/`writeText`; `memFs`'s `io` fake in `test-support.ts` likewise. Both MUST gain `readDir`/`exists` or every init test fails to typecheck.
- The example `haiku.workflow.ts` declares `harness: "copilot"` — do NOT reuse it; author all 8 templates fresh under `templates/` so `index.json[i].harness` matches each file (CI/validation invariant).

```ts
import { z } from "zod";
import { ok, err } from "neverthrow";
import type { Result } from "neverthrow";
import type { Immutable } from "@workflow/core";
import type { AppDeps } from "./app.js";

const Agents = z.union([z.number().int().nonnegative(), z.literal("N")]);
const TemplateEntry = z.object({
  name: z.string(),
  description: z.string(),
  harness: z.enum(["claude", "codex", "copilot", "raw-api"]),
  tags: z.array(z.string()).default([]),
  complexity: z.enum(["beginner", "intermediate", "advanced", "reference"]).optional(),
  whenToUse: z.string().optional(),
  agents: Agents.optional(),
  recommended: z.boolean().default(false),
  multiFile: z.boolean(),
  entry: z.string(),
  dir: z.string().optional(),
});
export type TemplateEntry = z.infer<typeof TemplateEntry>;
export const TemplateIndex = z.object({ version: z.literal(1), templates: z.array(TemplateEntry) });
export type TemplateIndex = z.infer<typeof TemplateIndex>;

export function loadTemplateIndex(
  deps: Pick<AppDeps, "env" | "io">,
): Result<Immutable<TemplateIndex>, string> {
  const raw = deps.io.readText(`${deps.env.templatesDir}/index.json`);
  if (raw === undefined) return err("templates index not found — reinstall defineworkflow");
  let json: unknown;
  try {
    json = JSON.parse(raw);
  } catch {
    return err("malformed templates/index.json: invalid JSON");
  }
  const parsed = TemplateIndex.safeParse(json);
  return parsed.success
    ? // oxlint-disable-next-line typescript/consistent-type-assertions -- ingress data exposed as deeply-readonly per the Immutable convention
      ok(parsed.data as Immutable<TemplateIndex>)
    : err(`malformed templates/index.json: ${parsed.error.message}`);
}
```

## Task 4: `init` command

**Files:**

- Create: `packages/cli/src/commands/init.ts`
- Test: extend `packages/cli/src/commands/commands.test.ts`

Behavior (helpers return `Result<…,string>`; handler maps err → `deps.ui.print("error: …\n")` + `return 1`):

1. No `template` → delegate to `listTemplatesCommand({ json: false }, deps)`; return its code (gallery-first).
2. `loadTemplateIndex` → find entry by `name`; not found → `error: unknown template '<x>' (run \`workflow list-templates\`)` + return 1.
3. Resolve harness: explicit `args.harness` (validate it's one of the 4; bad → error+1) else auto-detect:
   - declared in `deps.adapters.detected` → keep;
   - else first of `deps.adapters.detected` → use it (note substitution);
   - else `ANTHROPIC_API_KEY` in `deps.env.vars` → `"raw-api"`;
   - else keep declared (+ install hint later).
4. Destination dir = `args.dir ?? deps.env.cwd`.
5. Single-file: `destPath = <dir>/<name>.workflow.ts`. Multi-file: `destDir = <dir>/<name>/`, entry at `<destDir>/<entry>`.
6. Conflict: if `--dry-run` skip writes. Else if target exists and `!force`: on `isTTY` prompt via `deps.consent.io.question("…overwrite? [y/N]: ")` (only `y`/`yes` proceeds) else abort (return 1, print message). Non-TTY + exists + !force → `error: <path> exists (use --force)` + return 1.
7. Write: rewrite harness on the entry source only (`rewriteHarness`), `deps.io.writeText`. Multi-file: enumerate `deps.io.readDir(templatesDir/<dir>)`, copy each file (entry gets rewrite, others verbatim) into `destDir`.
8. `--dry-run`: print "would write: <paths>" + "harness: <chosen> (<reason>)"; write nothing; return 0.
9. Print stack-aware next steps (created line + 3 commands; harness-install hint only if nothing detected and no key).
10. Mock-run gate: `shouldRun = args.run || (deps.env.isTTY && !args.noRun && !args.yes && (await confirm))`. confirm via `deps.consent.io.question("Run it now with mocked agents (no tokens)? [Y/n]: ")` → not `n`/`no`. Non-TTY/`--yes` never prompt (run only if `--run`). If run → `return runCommand({ script: <entryDestPath>, detach: false, yes: true, mock: args.mode === "mock" }, deps)`.

`rewriteHarness(source, harness)`:

```ts
function rewriteHarness(source: string, harness: HarnessId): string {
  return source.replace(/(\bharness\s*:\s*)["'][^"']+["']/, `$1"${harness}"`);
}
```
